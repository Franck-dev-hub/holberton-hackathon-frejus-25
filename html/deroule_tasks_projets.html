Déroulement des Tasks et Projets : Le Cycle de Travail à Holberton

Le succès à Holberton School repose sur votre capacité à comprendre et à maîtriser le cycle de vie d'une tâche ou d'un projet. De la lecture du sujet à la validation finale, chaque étape est cruciale pour consolider vos connaissances et développer votre autonomie.

---

1. Comprendre le Sujet : La Première Étape Cruciale

Chaque tâche et projet débute par une description détaillée sur **Hippocampe**. C'est le cahier des charges de votre travail.

  • **Lecture Attentive :** Lisez le sujet plusieurs fois. Soulignez les mots clés, les contraintes, les objectifs et les exemples.
  • **Comprendre les Prérequis :** Assurez-vous d'avoir bien compris tous les concepts nécessaires avant de coder. Si un prérequis n'est pas clair, c'est le moment de poser des questions !
  • **Checkez Betty ! :** N'oubliez pas les exigences de style et de documentation spécifiées par Betty dès le début.
  • **Demandez si nécessaire :** Si un aspect du sujet n'est pas clair après relecture, demandez de l'aide sur Slack à vos pairs ou aux facilitateurs. Ne restez pas bloqué sans comprendre.
---

2. Développement Local : Codez, Testez, Itérez

Une fois le sujet compris, il est temps de passer à l'action sur votre environnement de développement.

  • **Clonez le Dépôt :** Récupérez les fichiers de base du projet depuis GitHub.
  • **Codez :** Écrivez votre code en respectant les spécifications. Faites des **commits fréquents** et avec des messages clairs.
  • **Testez Localement :** N'attendez pas de soumettre pour tester ! Utilisez des outils comme Betty et d'autres tests si fournis, ou créez les vôtres.
  • **Itérez :** Le développement est un processus itératif. Vous allez souvent écrire du code, le tester, trouver des erreurs, le corriger, et répéter. C'est normal !
---

3. Soumission et Validation : La Preuve de Votre Travail

C'est l'étape où votre travail est évalué par les systèmes de checks automatiques et/ou par vos pairs.

  • **`git add .` puis `git commit -m "Your descriptive message"` :** Enregistrez vos modifications.
  • **`git push` :** Envoyez votre code sur GitHub. Une fois poussé, Hippocampe va généralement déclencher des checks automatiques.
  • **Vérifiez les Checks Automatiques :** Accédez à Hippocampe pour voir les résultats de vos checks. Un résultat en vert signifie succès, en rouge signifie échec.
    • **Comprendre les Erreurs :** Si un check échoue, lisez attentivement le message d'erreur. C'est votre guide pour la correction.
    • **Correction et Re-soumission :** Corrigez votre code en fonction des erreurs, puis ajoutez, commitez et poussez à nouveau.
  • **Peer Review (Relecture par les pairs) :** Pour de nombreux projets, une fois les checks automatiques passés, votre travail sera soumis à la relecture d'autres étudiants.
    • **Soyez Constructif :** Lorsque vous révisez le code de vos pairs, soyez respectueux et donnez des retours clairs.
    • **Recevez le Feedback :** Acceptez le feedback sur votre propre code comme une opportunité d'apprendre et de vous améliorer.
---

4. Conseils pour Optimiser Votre Flux de Travail

  • **N'attendez Pas la Dernière Minute :** Commencez tôt ! Les problèmes imprévus arrivent souvent.
  • **Faites des Pauses :** Un esprit fatigué fait plus d'erreurs. Prenez de courtes pauses pour vous aérer.
  • **Communiquez :** Si vous êtes bloqué, demandez de l'aide sur Slack. Ne restez pas seul avec un problème.
  • **Relisez Votre Code :** Avant de soumettre, prenez un moment pour relire votre propre code. Vous pourriez détecter des erreurs simples.
  • **Gérez Votre Temps :** Apprenez à estimer le temps nécessaire pour chaque tâche et à prioriser.
