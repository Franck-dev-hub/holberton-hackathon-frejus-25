Betty : Règles de Style et Utilisation à Holberton School

Betty est un outil central dans l'écosystème Holberton...

1. Qu'est-ce que Betty et Pourquoi est-elle Importante ?

  • Cohérence : Assurer que tout le code produit suit les mêmes conventions...
  • Lisibilité : Un code bien formaté est plus facile à comprendre...
  • Bonnes Pratiques : Betty impose des pratiques robustes...
  • Préparation à l'Industrie : Maîtriser Betty vous prépare à suivre des standards professionnels.
  • Validation des Projets : Vos projets en C doivent passer Betty pour être validés.

2. Comment Utiliser Betty (Les Commandes Essentielles)

a. Vérifier un Fichier Spécifique

betty votre_fichier.c

b. Vérifier Tous les Fichiers C dans le Répertoire Courant

betty .
betty *.c *.h

c. Comprendre la Sortie de Betty

  • Nom du fichier
  • Numéro de ligne
  • Description de l’erreur

Exemple :
main.c:10: [Whitespace/Indentation] missing space after comma
_putchar.c:5: [LineLength] line is too long (85 > 80)

3. Astuces et Erreurs Fréquentes avec Betty

  • Vérifiez Souvent : Ne pas attendre la fin pour lancer Betty.
  • Lisez les Messages : Ils sont précis et utiles.
  • Lignes Vides : Évitez celles qui sont inutiles.
  • Indentation : Configurez votre éditeur pour 4 espaces.
  • Commentaires : Respectez le style Holberton.
  • Lignes Trop Longues : Limitez à 80 caractères.
  • Variables Non Utilisées : Supprimez-les.

Betty est votre amie, même si elle peut sembler exigeante au début...
