Checks : Comment ca fonctionne ?

1. Qu'est-ce qu'un Check et Comment ça Fonctionne ?

  • Conformité Betty : Vérification du style et des bonnes pratiques...
  • Fonctionnalité : Est-ce que votre code fonctionne ?
  • Contraintes Spécifiques : Respect des règles spécifiques au projet.
  • Documentation : Présence et qualité du README, commentaires, etc.

Ces checks sont déclenchés automatiquement après un git push vers GitHub...

2. Lire et Interpréter les Résultats des Checks sur Hippocampe

  • Ligne Verte : Le test est passé avec succès.
  • Ligne Rouge : Le test a échoué.
  • Messages d'Erreur : Très utiles pour comprendre l'origine de l'échec.
    • Type d’erreur (Output, Betty, Memory...)
    • Fichier concerné, ligne, fonction
    • Ex : "ligne trop longue", "valeur incorrecte"…
  • Exemple : 
    [+] test 0: main.c - OK
[x] test 1: output_check.c - Failed
    > Expected "Hello, Holberton\n" but got "Hello Holberton\n"
[x] test 2: betty_check.c - Failed
    > main.c:5: [LineLength] line is too long (85 > 80)

3. Corriger et Re-soumettre : Le Cycle Essentiel

  1. Analysez l'Échec : Comprenez le message.
  2. Localisez le Problème : Trouvez la source du bug.
  3. Corrigez le Code : Faites les ajustements nécessaires.
  4. Testez Localement : Avant tout nouveau push.
  5. Ajoutez, Commitez, Poussez : 
    git add .
git commit -m "fix: Resolve output_check error for task 1"
git push
  6. Re-vérifiez : Sur Hippocampe après le push.

Conseils pour Maîtriser les Checks

  • Ne Gagnez Pas de Temps : Les checks sont là pour vous faire progresser.
  • Posez des Questions : Si vous ne comprenez pas un échec.
  • Documentez-vous : Lisez la doc, relisez les consignes.
  • Petits Commits : Pour faciliter le debug.
